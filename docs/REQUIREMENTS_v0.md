# Today.bike 웹사이트 요구사항 문서

> 버전: 0.1 (초안)
> 작성일: 2026-03-11

---

## 1. 프로젝트 개요

**Today.bike**는 자전거 분해정비·고장수리 전문 샵으로, 투명하고 집착에 가까운 수준의 정비 서비스를 제공한다.
현재 엑셀로 관리 중인 고객별 정비이력·피팅정보·입출고이력을 전용 웹사이트로 전환하여, 고객 신뢰도를 높이고 샵 운영 효율을 극대화하는 것이 목표다.

---

## 2. 핵심 가치

- **투명성** — 정비 진행 현황을 고객이 실시간으로 확인할 수 있다
- **기록** — 모든 정비·파츠·피팅 이력이 영구 보존된다
- **신뢰** — 분해 전/후 사진, 교체 파츠 정보를 공개하여 고객 신뢰를 얻는다

---

## 3. 사용자 유형

| 유형 | 설명 |
|------|------|
| **관리자 (샵 오너)** | 전체 고객·자전거·정비 데이터 관리, 블로그 포스트 작성, 파츠 판매 관리 |
| **고객** | 본인 자전거의 정비이력·현황 조회, 견적 확인 |
| **방문자** | 샵 소개, 서비스 안내, 블로그 포스트 열람, 파츠 쇼핑 |

---

## 4. 주요 기능 요구사항

### 4-1. 고객 & 자전거 관리 (관리자)

- [ ] 고객 등록·수정·삭제 (이름, 연락처, 메모)
- [ ] 고객별 자전거 등록 (브랜드, 모델, 프레임 사이즈, 시리얼, 구매일, 사진)
- [ ] 고객 검색 및 필터링

### 4-2. 정비 이력 관리 (핵심)

| 이력 유형 | 상세 내용 |
|-----------|-----------|
| **분해정비** | 입고일, 작업 단계별 진행 상황, 전·후 사진, 완료일, 비용 |
| **고장수리** | 증상, 원인 진단, 수리 내용, 교체 파츠, 비용 |
| **파츠 교체** | 교체 전·후 파츠 정보, 브랜드, 모델, 가격 |
| **업그레이드** | 기존 스펙 → 변경 스펙, 비용 |
| **기변 (프레임 교체)** | 기존 프레임 → 신규 프레임, 이전 파츠 이관 내역 |

### 4-3. 피팅 정보 관리

- [ ] 고객별 바이크 피팅 데이터 기록 (포지션 수치, 클릿 세팅, 안장 높이/전후 등)
- [ ] 피팅 히스토리 (날짜별 변경 내역)
- [ ] 피팅 전·후 사진 첨부

### 4-4. 입출고 이력 관리

- [ ] 자전거 입고 등록 (날짜, 고객, 작업 종류, 메모)
- [ ] 출고 처리 (출고일, 최종 확인 사항, 고객 사인/확인)
- [ ] 현재 입고 중인 자전거 목록 (작업 현황 보드)

### 4-5. 고객 전용 포털

- [ ] 고객 로그인 (전화번호 또는 이메일)
- [ ] 본인 자전거 목록 조회
- [ ] 정비이력 타임라인 (날짜순 전체 이력)
- [ ] 현재 진행 중인 정비 현황 실시간 확인
- [ ] 분해정비 단계별 사진 확인 (투명성 제공)
- [ ] 피팅 데이터 조회

### 4-6. 블로그 / 정비 후기

- [ ] 관리자 블로그 포스트 작성·편집·삭제
- [ ] 카테고리 분류 (분해정비, 고장수리, 업그레이드, 피팅, 기타)
- [ ] 네이버 블로그 기존 포스트 마이그레이션 (수동 입력 또는 임포트)
- [ ] 이미지 다중 업로드 지원
- [ ] 방문자 열람 (SEO 최적화)

### 4-7. 파츠 & 부품 판매

- [ ] 파츠 상품 등록 (이름, 브랜드, 카테고리, 가격, 재고, 이미지)
- [ ] 카테고리 관리 (구동계, 휠셋, 안장, 핸들바, 페달 등)
- [ ] 장바구니 및 주문 기능 (1단계: 문의/견적 요청으로 시작 가능)
- [ ] 재고 관리

## 5. 비기능 요구사항

| 항목 | 내용 |
|------|------|
| **플랫폼** | 웹 (모바일 반응형 필수 — 고객이 스마트폰으로 이력 확인) |
| **접근성** | 고객 포털은 로그인 필요, 블로그·샵 소개는 공개 |
| **이미지 관리** | 정비 사진 다수 업로드·저장 가능 (스토리지 고려) |
| **성능** | 이력 조회 빠른 응답 (검색·필터) |
| **보안** | 고객 개인정보 보호, 관리자 권한 분리 |

## 6. 정보 구조 (사이트맵 초안)

```
today.bike
├── 홈 (샵 소개, 서비스 특징)
├── 서비스 안내
│   ├── 분해정비
│   ├── 고장수리
│   ├── 피팅
│   └── 업그레이드
├── 정비 후기 (블로그)
├── 파츠 샵
├── 고객 포털 (로그인)
│   ├── 내 자전거
│   ├── 정비 이력
│   ├── 피팅 정보
│   └── 현재 진행 현황
└── 관리자 (로그인)
    ├── 고객 관리
    ├── 자전거 관리
    ├── 정비 관리
    ├── 입출고 관리
    ├── 블로그 관리
    └── 파츠 관리
```

## 7. 개발 우선순위 (단계별)

### Phase 1 — 핵심 운영 도구
> 엑셀 대체가 목적. 샵 내부 운영 효율화

- 고객·자전거 등록
- 정비이력 기록 (분해정비, 수리, 파츠교체)
- 입출고 관리
- 관리자 대시보드

### Phase 2 — 고객 신뢰 구축
> 투명성 제공

- 고객 포털 (이력 조회, 현황 확인)
- 분해정비 단계별 사진 공유

### Phase 3 — 마케팅 & 매출 확장
> 외부 노출 및 수익화

- 블로그 (정비 후기)
- 네이버 블로그 마이그레이션
- 파츠 판매 기능
